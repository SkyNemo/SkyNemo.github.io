使用Mycat1.6.7.4对MySQL5.7进行读写分离配置

简介

MySQL读写分离，原理是让Master数据库处理事务性增、删除、修改、更新操作（CREATE、 INSERT、UPDATE、DELETE），而让Slave数据库处理SELECT操作。所以，读写分离是基于主从复制完成的，这样可以保证在Master上修改数据，Slave同步之后，应用可以读取到Slave端 的数据

所有主机操作系统均为 CentOS7 2009 minimal install
